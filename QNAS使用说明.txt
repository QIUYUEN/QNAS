QNAS使用说明：

1. 在使用前，为了确保QNAS顺利运行，您的系统需要先安装Node.js，这是一个JavaScript运行环境，如果您的系统已经有Node.js，那么可以略过次步骤。

2. 双击根目录里的start.bat，会弹出一个命令窗口，这样QNAS就已经运行完成，注意，在NAS系统运行期间，请勿关闭这个命令窗口。

3. 接下来打开浏览器，输入http://localhost:8088，即可打开QNAS系统登录页面，QNAS系统默认使用8088端口，但如果您的8088端口无法启用，系统也会自动选择其他端口，您可以在日志文件logs\startup.log里查询使用的是哪个端口，然后把http://localhost:后面的8088改成该端口即可打开登录页面。

4. 现在，您局域网的设备，比如手机和平板电脑，都可以通过访问http://localhost:8088，进入到你作为NAS的电脑，对这台电脑的磁盘文件的进行访问和管理。

5. 系统默认只有一个管理员用户，用户名：admin，密码：admin123。登录后，您可以更改用户头像，也可以新增其他用户，点击页面左侧的用户名，然后在用户管理里添加新的账户，并赋予这些账户权限。

6. 开启外网访问功能，您需要将tools文件夹里的cloudflared.exe解压出来，放在tools文件夹的根目录里。

然后在QNAS系统页面左侧下方有基本的访问地址信息，如果您需要从外网访问这个NAS，那么请点击开启外网访问按钮，然后就可以获得一个外网地址，通过这个地址即可从外网访问到这台NAS。

QNAS的外网访问功能是通过Cloudflare加密隧道实现的，如果您NAS的电脑重启和关机，或QNAS系统的重启，都需要再次执行这个操作，以获得新的外网访问地址。同时也是由于加密隧道的原因，文件传输会比较慢，不太适合大容量文件的上传和下载，但它有效解决了NAS能够被外网访问的问题。
通常而言，只要您的NAS电脑不关机，QNAS一直稳定运行，这个地址就能一直被正常访问。
当您在外网能够正常访问NAS的时候，不要随意点击开启外网访问的按钮，这样它会生成一个新地址，而您的旧地址会立即失效无法再访问NAS，此时你必须在局域网登录进QNAS系统才能获知新地址。

7. 我们都知道数据无价，当您对文件进行删除操作的时候，它会被彻底删除，所以请谨慎操作。

8. QNAS的开发宗旨是打造一个人人都能拥有的轻量化NAS系统，没有太多繁琐操作和设置，即可实现在Windows里一样的文件管理操作。

QNAS使用説明：

1. 使用前に、QNASを正常に動作させるために、システムにNode.jsをインストールする必要があります。Node.jsはJavaScriptの実行環境です。すでにインストールされている場合は、この手順はスキップできます。
2. ルートディレクトリにある start.bat をダブルクリックすると、コマンドウィンドウが表示されます。これでQNASは起動完了です。NAS動作中は、このコマンドウィンドウを閉じないでください。
3. 次にブラウザを開き、http://localhost:8088 を入力するとQNASシステムのログインページが開きます。QNASはデフォルトで8088ポートを使用しますが、もし8088ポートが使用できない場合は、システムが自動的に別のポートを選択します。その場合は logs\startup.log を確認して使用されているポート番号を調べ、http://localhost: の後ろの8088をそのポート番号に変更してください。
4. これで、同じLAN内のデバイス（スマートフォンやタブレットなど）から http://localhost:8088 にアクセスすることで、このNASとして動作しているPCに接続し、ディスク内のファイルを閲覧・管理できます。
5. システムの初期ユーザーは管理者のみです。ユーザー名：admin、パスワード：admin123 です。ログイン後、ユーザーのアバターを変更したり、新しいユーザーを追加できます。左側のユーザー名をクリックし、ユーザー管理画面から新規アカウントを作成し、権限を付与できます。
6. 外部ネットワークアクセスを有効にするには、toolsフォルダ内の cloudflared.exe を解凍し、toolsフォルダのルートに配置してください。

その後、QNASシステム画面の左下に基本アクセス情報が表示されます。外部からこのNASにアクセスしたい場合は、「外部アクセスを有効化」ボタンをクリックしてください。すると外部アクセス用のURLが生成され、そのURLを使って外部ネットワークからNASにアクセスできます。

QNASの外部アクセス機能はCloudflareの暗号化トンネルを利用しています。そのため、NASの再起動や電源オフ、QNASの再起動後には再度この操作を実行し、新しい外部アクセスURLを取得する必要があります。また、暗号化トンネルの特性上、ファイル転送速度は遅く、大容量ファイルのアップロードやダウンロードにはあまり適していませんが、NASを外部からアクセス可能にするという目的は十分に満たします。

通常、NASのPCが電源オフにならずQNASが安定して動作している限り、このURLは継続して使用できます。

外部アクセスが有効な状態で、むやみに「外部アクセスを有効化」ボタンを押さないでください。新しいURLが生成されると、以前のURLは即座に無効になります。その場合は、ローカルネットワークからQNASにログインして新しいURLを確認する必要があります。

7. データは非常に重要です。ファイルを削除すると完全に削除されるため、慎重に操作してください。
8. QNASの開発目的は、誰でも簡単に使える軽量NASシステムを作ることです。複雑な操作や設定を必要とせず、Windowsのように直感的にファイル管理ができることを目指しています。

QNAS User Guide:

1. Before using QNAS, you need to install Node.js on your system to ensure it runs properly. Node.js is a JavaScript runtime environment. If it is already installed on your system, you can skip this step.
2. Double-click the start.bat file in the root directory. A command window will pop up, indicating that QNAS is running successfully. Do not close this window while the NAS system is running.
3. Next, open your browser and enter http://localhost:8088 to access the QNAS login page. By default, QNAS uses port 8088. If port 8088 is unavailable, the system will automatically select another available port. In that case, check logs\startup.log to find the port being used, then replace 8088 in http://localhost: with the correct port number.
4. At this point, devices on your local network (such as phones and tablets) can access http://localhost:8088 to connect to this NAS computer and manage files on its disks.
5. The system has only one default administrator account. Username: admin, Password: admin123. After logging in, you can change the user avatar or add new users. Click the username on the left side of the page, then go to user management to create new accounts and assign permissions.
6. To enable external access, extract cloudflared.exe from the tools folder and place it in the root directory of the tools folder.

Then, basic access information will be shown at the bottom left of the QNAS interface. If you want to access this NAS from outside the local network, click the “Enable External Access” button. A public external URL will be generated, which you can use to access the NAS remotely.

The external access feature of QNAS is implemented using a Cloudflare encrypted tunnel. Therefore, if the NAS PC is restarted or shut down, or if the QNAS system is restarted, you need to repeat this process to obtain a new external access URL. Due to the nature of encrypted tunneling, file transfer speeds are relatively slow and not suitable for large file uploads or downloads, but it effectively enables external access to the NAS.

Generally, as long as the NAS computer remains powered on and QNAS is running stably, this URL will remain valid.

When you already have external access working, do not click the “Enable External Access” button again unnecessarily. Doing so will generate a new URL, and the previous URL will immediately become invalid. In that case, you must log into QNAS from the local network to retrieve the new address.

7. Data is extremely valuable. When you delete files, they are permanently removed, so please operate with caution.
8. The goal of QNAS is to create a lightweight NAS system that anyone can use. It is designed to avoid complex operations and settings, allowing intuitive file management similar to Windows.

8. The development goal of QNAS is to create a lightweight NAS system that everyone can own. Without complicated operations or settings, it allows you to manage files just like you would in Windows.
